# job-crm LESSONS — 실수 & 재발방지

> 삽질할 때마다 여기 남긴다. 목적은 자책이 아니라 **같은 실수 반복 방지**.
> 형식: 증상 → 원인 → 해결 → 재발방지 규칙.
> Claude Code에게 작업 시킬 때 이 파일을 먼저 읽게 하면 과거 실수를 피할 수 있다.

---

## 템플릿 (복사해서 사용)

### [YYYY-MM-DD] 한 줄 제목
- **증상:** 무슨 일이 일어났나
- **원인:** 진짜 원인 (표면 원인 말고 근본 원인)
- **해결:** 어떻게 고쳤나
- **재발방지:** 다음엔 뭘 다르게 할까 (체크리스트/규칙으로)

## 기록

### [2026-07-01] GitHub README 켜고 만들면 push 충돌
- **증상:** README 켜고 원격 레포 생성 시, 로컬 create-next-app과 히스토리가 갈려 push 거부 위험
- **원인:** 원격에 초기 커밋(README)이 생겨 로컬과 공통 조상이 없음 (unrelated histories)
- **해결:** 레포 생성 시 README/​.gitignore/license 전부 off로 빈 레포 생성
- **재발방지:** 새 레포는 항상 "완전 빈 상태"로 만들고 .gitignore는 create-next-app이 생성하게 둔다
